Enfoque de PythonOS

Mi enfoque se basa en un subsistema ligero y minimalista, donde el usuario sienta una experiencia diferente a la de su sistema y pueda tener una vista más amplia del subsistema haciendo la instalación y usando la interfaz de consola.

A diferencia de otros proyectos que también buscan hacer microSO en python, PythonOS de kev402 tiene un estilo más simplicista y manual como algunos sistemas Linux. Además busca la ligereza usando python y el entorno del sistema operativo del usuario para complementar la ejecucción, haciendo que no sea necesario usar tanto código en la implementación, además tenemos un modelos más orientado en ayudar a instalar dependencias tanto en el instalador como en PythonOS comot tal.

PythonOS en Linux

De momento PythonOS solo está disponible en Windows, y esa versión no es compatible con Linux ya que maneja rutas con la unidad C:/ y tiene características adaptadas a Windows como el uso de pip global, pues en Linux se debe manejar las rutas relacionadas con / y se debe manejar un entorno virtual para evitar romper el sistema instalando forzosamente paquetes con pip, es de preferencia usar pip en vez de usar los gestores de paquetes de las distro de Linux ya que así se puede tener un manejo más general sin necesidad de hacer una versión para cada distro que maneje pacman, apt o dnf.

De momento la versión en Linux aún está en desarrollo, ya se están ajustando las rutas y haciendo que en el proceso del instalador se cree un entorno virtual y que los archivos esten directamente ahí.
